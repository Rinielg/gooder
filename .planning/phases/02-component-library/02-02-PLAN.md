  <action>
    Create a convenience `FormField` wrapper component that integrates React Hook Form with the shadcn `form.tsx` primitives and enforces the user's validation UX decisions.

    **File: `src/components/ui/form-field.tsx`**

    This component wraps shadcn's `FormItem`, `FormLabel`, `FormControl`, `FormMessage` to provide a consistent field layout. It does NOT replace shadcn's form.tsx -- it composes it.

    **Props interface:**
    ```typescript
    interface FormFieldProps {
      name: string;           // Field name matching the Zod schema key
      label: string;          // Label text displayed above the field
      description?: string;   // Optional help text below the field
      children: React.ReactNode; // The actual input element (Input, Select, Textarea, etc.)
    }
    ```

    **Validation UX decisions (LOCKED -- implement exactly):**
    1. Errors appear on blur -- use `mode: "onBlur"` when creating the form (document this in a code comment for consumers)
    2. Once an error is shown, it clears in real-time as the user fixes the input -- use `reValidateMode: "onChange"` (document this too)
    3. Inline error messages: small red text directly below the invalid field -- this is the default `FormMessage` behavior, just ensure it renders with `text-destructive text-sm`
    4. Error summary banner pattern: Create an exported `FormErrorSummary` component that reads the form error count from `useFormContext` and renders a banner like "3 fields need attention" when `errors` has entries. This is used at the top of forms on submit.

    **FormErrorSummary component (in same file):**
    ```typescript
    interface FormErrorSummaryProps {
      className?: string;
    }
    ```
    - Uses `useFormState` from react-hook-form to read `errors` object
    - Counts the number of error entries: `Object.keys(errors).length`
    - If count > 0: renders a styled banner with `bg-destructive/10 text-destructive border border-destructive/20 rounded-lg p-3` showing "{N} field(s) need attention"
    - If count === 0: renders nothing (returns null)
    - Includes an `AlertCircle` icon from lucide-react

    **Export both components:** `FormField` and `FormErrorSummary`

    **Usage pattern (document in a code comment at the top of the file):**
    ```typescript
    // Usage:
    // const form = useForm<z.infer<typeof schema>>({
    //   resolver: zodResolver(schema),
    //   mode: "onBlur",           // Errors appear on blur
    //   reValidateMode: "onChange" // Errors clear in real-time
    // });
    //
    // <Form {...form}>
    //   <FormErrorSummary />
    //   <FormField name="email" label="Email">
    //     <Input placeholder="you@example.com" />
    //   </FormField>
    // </Form>
    ```

    **Important:** The shadcn `form.tsx` (installed in 02-01) already provides `FormField` as a re-export of react-hook-form's `Controller`. To avoid naming conflicts, name this convenience wrapper `FormFieldWrapper` internally but export it as `{ FormFieldWrapper, FormErrorSummary }`. Alternatively, if shadcn's `FormField` is just a re-export, consider naming our wrapper `FormInput` or checking what shadcn's form.tsx actually exports and avoid collision.

    Read `src/components/ui/form.tsx` first to understand its exports, then name accordingly to avoid conflicts.

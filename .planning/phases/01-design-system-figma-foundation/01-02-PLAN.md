    **Step 1: Update layout.tsx to initialize Geist fonts**

    Add Geist font imports and apply CSS variables on the html element. Keep all existing functionality (Metadata, Toaster).

    ```typescript
    import type { Metadata } from "next";
    import { GeistSans } from "geist/font/sans";
    import { GeistMono } from "geist/font/mono";
    import { Toaster } from "sonner";
    import "./globals.css";

    export const metadata: Metadata = {
      title: "Brand Voice Platform",
      description: "AI-powered brand voice management and content generation",
    };

    export default function RootLayout({
      children,
    }: Readonly<{
      children: React.ReactNode;
    }>) {
      return (
        <html lang="en" className={`${GeistSans.variable} ${GeistMono.variable} light`}>
          <body className="font-sans antialiased">
            {children}
            <Toaster position="top-right" richColors />
          </body>
        </html>
      );
    }
    ```

    The key changes:
    - Import GeistSans and GeistMono from the geist package
    - Apply both `.variable` CSS class names on `<html>` (this creates `--font-geist-sans` and `--font-geist-mono` CSS variables)
    - Keep `className="light"` (light mode only per project decision)
    - Keep `font-sans antialiased` on body (Tailwind will resolve `font-sans` to Geist via config)

    **Step 2: Update tailwind.config.ts with font family, spacing grid, and shadow elevation**

    Add `defaultTheme` import and extend the theme. Keep ALL existing config (darkMode, content, colors, borderRadius, keyframes, animation, plugins).

    Add to `theme.extend`:

    1. **fontFamily** (DSGN-02):
    ```typescript
    fontFamily: {
      sans: ["var(--font-geist-sans)", ...defaultTheme.fontFamily.sans],
      mono: ["var(--font-geist-mono)", ...defaultTheme.fontFamily.mono],
    },
    ```

    2. **boxShadow** (DSGN-04) - semantic elevation system:
    ```typescript
    boxShadow: {
      'elevation-1': '0 1px 3px 0 rgb(0 0 0 / 0.08), 0 1px 2px -1px rgb(0 0 0 / 0.08)',
      'elevation-2': '0 4px 6px -1px rgb(0 0 0 / 0.08), 0 2px 4px -2px rgb(0 0 0 / 0.08)',
      'elevation-3': '0 10px 15px -3px rgb(0 0 0 / 0.08), 0 4px 6px -4px rgb(0 0 0 / 0.08)',
      'elevation-4': '0 20px 25px -5px rgb(0 0 0 / 0.1), 0 8px 10px -6px rgb(0 0 0 / 0.1)',
    },
    ```

    Elevation mapping (document in a comment above the object):
    - elevation-1: Cards, containers (resting state)
    - elevation-2: Dropdowns, popovers, hover states
    - elevation-3: Modals, dialogs, overlays
    - elevation-4: Floating action elements

    3. **spacing** (DSGN-03) - add 8px grid semantic tokens:

    Do NOT override theme.spacing. Use theme.extend.spacing to ADD grid-aligned tokens alongside defaults. This avoids breaking existing component internals (py-0.5, px-2.5, etc. are intentional in shadcn components).

    ```typescript
    spacing: {
      '4.5': '1.125rem',  // 18px
      '13': '3.25rem',    // 52px
      '15': '3.75rem',    // 60px
      '18': '4.5rem',     // 72px
      '22': '5.5rem',     // 88px
      '26': '6.5rem',     // 104px
      '30': '7.5rem',     // 120px
    },
    ```

    NOTE on spacing approach: Tailwind 3's default spacing scale already provides 8px-aligned values at commonly-used sizes (p-2=8px, p-4=16px, p-6=24px, p-8=32px, etc.). The existing defaults ARE the 8px grid for most practical purposes. We only need to add the GAPS in the scale that might be useful for layout work. The 8px grid is "enforced" by convention (using these values for layout spacing) rather than by removing non-grid values (which would break shadcn component internals).

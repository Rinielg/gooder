## Decisions Made

- `isCurrentlyStreaming` gates on both `"streaming"` AND `"submitted"` status — `"submitted"` is the pre-first-token phase where content is still partial. Both states use MemoizedMarkdown.
- `AdjustDialog` placed at `z-20` (above input `z-10`) — ensures it appears above the chat input form without conflict.
- `submitAdjust()` truncates `sectionBody` to first 200 characters — avoids echoing full content in every adjust prompt, keeping chat history readable (research pitfall 7).
- `lastAssistantId` computed via `useMemo` on `displayMessages` — avoids recomputing on every render from unrelated state changes.
- WhatsApp card: removed conditional `MemoizedMarkdown` fallback — `section.body.trim()` is always sufficient to populate the styled Message field.
- `parseScreenSections()` uses multi-variant regex (e.g., `headline|heading|title|screen title`, `cta|call to action|button|primary action`) to be robust against varying AI output formats.
- Accordion default-collapsed: score cards are supplementary information; users should opt in to expand rather than having them auto-open.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] WhatsApp card rendered gray MemoizedMarkdown when labeled fields not found**
- **Found during:** Post-checkpoint human verification (Task 3 approved)
- **Issue:** When AI WhatsApp response body had no labeled `message:` or `body:` field, the card fell through to a gray MemoizedMarkdown block — visually inconsistent with all other card types
- **Fix:** Replaced conditional `text ? <div>...</div> : <MemoizedMarkdown>` with unconditional `<div>` using `fields.message ?? fields.body ?? section.body.trim()` as text value
- **Files modified:** `src/app/(dashboard)/chat/output-card.tsx`
- **Committed in:** `f194ba1`

**2. [Rule 1 - Bug] UX Journey screen cards showed raw markdown blob instead of structured fields**
- **Found during:** Post-checkpoint human verification (Task 3 approved)
- **Issue:** JourneyScreenCard rendered all screen content as a single MemoizedMarkdown block — Headline, Body copy, and CTA were not visually separated into labelled sections despite AI generating structured content
- **Fix:** Added `parseScreenSections()` function with regex extraction covering common field naming variants; restructured JourneyScreenCard with visual header strip (bg-accent/40) and conditional Headline/Body copy/CTA section rendering; unstructured content falls back to markdown
- **Files modified:** `src/app/(dashboard)/chat/output-card.tsx`
- **Committed in:** `f194ba1`

**3. [Rule 1 - Bug] Adherence score accordion auto-expanded on every score arrival**
- **Found during:** Post-checkpoint human verification (Task 3 approved)
- **Issue:** `fetchAdherenceScore()` in page.tsx called `setExpandedIds((prev) => new Set(prev).add(messageId))` — score card opened automatically on every AI response which was disruptive to UX; dimension rows inside score-card.tsx also started fully expanded
- **Fix:** Removed `setExpandedIds` call from `fetchAdherenceScore()`; moved `expandedDims` and `showAll` state declarations after `sorted` array derivation (fixing variable ordering); both default to collapsed/hidden
- **Files modified:** `src/app/(dashboard)/chat/page.tsx`, `src/app/(dashboard)/chat/score-card.tsx`
- **Committed in:** `f194ba1`

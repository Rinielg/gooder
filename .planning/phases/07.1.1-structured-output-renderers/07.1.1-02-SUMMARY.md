## Decisions Made

- **extractJSON() preprocessing:** Output Agent wraps JSON responses in ` ```json ``` ` code fences. Rather than fixing the Output Agent prompt, added a client-side stripping helper so the renderer is robust to both fence-wrapped and bare JSON responses.
- **Inline typeMap in onAdjust:** The channel type → DetectedOutputType mapping is a small, local concern; an inline Record avoids needing to export another constant from output-card.tsx.
- **isCurrentlyStreaming gate untouched:** The gate `(status === "streaming" || status === "submitted") && message.id === lastAssistantId` was intentionally preserved exactly as-is per plan instructions.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Added extractJSON() to fix renderer fallthrough on code-fence-wrapped JSON**

- **Found during:** Task 2 (human verification checkpoint) — post-verify
- **Issue:** Output Agent was returning JSON wrapped in ` ```json ... ``` ` markdown code fences. `isValidJSON()` checked `str.trim().startsWith("{")` which fails when the string starts with a backtick, causing the renderer to fall through to the `MemoizedMarkdown` raw text display path rather than the `StructuredOutputRenderer` path.
- **Fix:** Added `extractJSON(str: string): string` helper that strips ` ```json ``` ` or ` ``` ``` ` fences before the validity check and JSON.parse. Updated `isValidJSON()`, the renderer IIFE `JSON.parse`, and the `saveOutput` type-detection `JSON.parse` to all call `extractJSON()` first.
- **Files modified:** `src/app/(dashboard)/chat/page.tsx`
- **Verification:** Confirmed fix resolves fallthrough; structured output renders in typed visual preview cards as expected.
- **Committed in:** `bdf0b21` (post-verify fix commit)

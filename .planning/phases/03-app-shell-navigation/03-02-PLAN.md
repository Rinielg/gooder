<task type="auto">
  <name>Task 2: Create dynamic Breadcrumbs component with path parsing</name>
  <files>
    src/components/layout/breadcrumbs.tsx
  </files>
  <action>
    **Create `src/components/layout/breadcrumbs.tsx`:**
    - `'use client'` directive (uses `usePathname` hook)
    - Import: `usePathname` from `next/navigation`
    - Import: `Link` from `next/link`
    - Import: `Breadcrumb, BreadcrumbList, BreadcrumbItem, BreadcrumbLink, BreadcrumbPage, BreadcrumbSeparator` from `@/components/ui/breadcrumb`

    **Props interface:**
    ```typescript
    interface BreadcrumbsProps {
      /** Override labels for specific path segments. Keys are URL segments, values are display names. */
      overrides?: Record<string, string>
    }
    ```

    **Implementation:**
    - Get current pathname via `usePathname()`
    - Split pathname by `/` and filter out empty strings to get segments array
    - Define default segment label mapping:
      ```typescript
      const defaultLabels: Record<string, string> = {
        chat: 'Chat',
        profiles: 'Profiles',
        standards: 'Standards & Rules',
        objectives: 'Objectives',
        definitions: 'Definitions',
        outputs: 'Output Library',
        settings: 'Settings',
        train: 'Training',
      }
      ```
    - Merge `overrides` prop into default labels (overrides take precedence)
    - For each segment, build cumulative href: `'/' + segments.slice(0, index + 1).join('/')`
    - Resolve display label: check merged labels map first, then use the raw segment (this handles dynamic `[id]` segments -- the parent page should pass overrides with the entity name)
    - If only 1 segment (top-level page like `/chat`), return `null` -- no breadcrumbs needed on top-level pages (per user decision: "breadcrumbs on nested pages")
    - Render using shadcn Breadcrumb primitives:
      - Each non-last segment: `BreadcrumbItem` > `BreadcrumbLink` with `asChild` wrapping `Link`
      - Last segment: `BreadcrumbItem` > `BreadcrumbPage` (non-clickable, current page)
      - `BreadcrumbSeparator` between items (default ChevronRight icon from breadcrumb.tsx)
    - Use proper React fragment or array mapping -- avoid wrapping separators in extra `<div>` elements (use `React.Fragment` with key)

    **Named export: `Breadcrumbs`**

    **Key design note:** For dynamic route segments like `/profiles/[id]`, the overrides prop allows parent pages to pass `{ 'abc-123-uuid': 'Brand Name' }` so the breadcrumb shows the entity name instead of the raw ID. This avoids fetching data inside the breadcrumb component.
  </action>

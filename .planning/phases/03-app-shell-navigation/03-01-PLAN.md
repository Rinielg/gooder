  <action>
    **Create `src/hooks/use-sidebar-state.ts`:**
    - Custom hook managing `collapsed` boolean state
    - Uses `useState(false)` as default (expanded)
    - Uses `useEffect` to read from `localStorage.getItem('sidebar-collapsed')` on mount -- NEVER read localStorage during render (causes hydration mismatch)
    - Tracks `mounted` boolean via `useEffect` -- starts false, set true after first mount
    - Second `useEffect` persists `collapsed` to `localStorage.setItem('sidebar-collapsed', String(collapsed))` only when `mounted` is true
    - localStorage key: `'sidebar-collapsed'`
    - Returns `{ collapsed, setCollapsed, mounted }`
    - Export named: `useSidebarState`

    **Create `src/hooks/use-media-query.ts`:**
    - Custom hook accepting a `query` string parameter (CSS media query)
    - Uses `useState(false)` as default
    - Uses `useEffect` to call `window.matchMedia(query)`, read initial `.matches`, add `'change'` event listener, clean up on unmount
    - Tracks `mounted` boolean -- returns `false` before mount to prevent SSR hydration mismatch
    - Returns `mounted ? matches : false`
    - Export named: `useMediaQuery`

    **Refactor `src/components/layout/app-sidebar.tsx`:**
    - Replace internal `useState(false)` for collapsed with `useSidebarState()` hook import
    - Keep `'use client'` directive
    - Import from `@/hooks/use-sidebar-state`
